Use cases of null and undefined

Null and undefined are often misunderstood as the same thing. They are both falsy value, but there are some differences between nulll and undefined.

Undefined means absence of value whereas null itself is a value.

Use Cases of Undefined:

  • When a varibale is declared without initialization, it automatically become undefined.
  • When a var is hoisted, it only hoist the variable undefined.
  • When we do not return anything from a function, it returns undefined.
  • When we try to access a object property which is not available, it returns undefined.

Use Cases of Null:

  • Null can be used to remove everything from a variable.
  • We can use null to initialize a variable
