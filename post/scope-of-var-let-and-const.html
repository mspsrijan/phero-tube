Scope of var, let, and const

Scopes can be a little wierd thing when you come to learn JavaScript from a different programming language.

Before ES6, Javascript had two types of scope- Function Scope and Global Scope. And there was only only variable type- var.

Var is global scoped and function scoped. When a var is declared inside a function, that var is only accessible inside that function. But if a var is declared anywhere outside of a function scope whether it is in global scope or in a block (i.e. in if-else block, for loop etc), it is accessible from anywhere. For this nature, var poseess the risk of being updated or replaced accidentally.

To fix this issue, ES6 introduced a new scope called Block scope and two more variable type- let and const.

Let and const are both block scoped, functions scoped and global scoped. That means in addition to vars characteristics, let and const are exclusive when they are declared inside of a block that is wrapped with curly braces.

Additionally let and const cannot be redeclared like var. So these two fixes the issues var has.
